What is a shell?

A Unix shell is both a command interpreter and a programming language, providing an interface to system utilities and a language to compose them into repeatable workflows [1-6]. Shells can run interactively by reading from a terminal or non‑interactively by reading from a file or string, enabling both ad‑hoc use and automation [1-7]. Bash, ksh, and csh are different shells with overlapping purposes and varying features and syntax.

What you’ll learn

  • Define a shell and Bash, and understand interactive vs non‑interactive modes
  • Run common commands, check your Bash version, and discover the built‑in help system
  • Write and execute a first script with the right shebang and permissions, and know why ./script.sh is required
  • Grasp exit status, pipelines, redirection, and the evaluation steps Bash applies to input
  • Use time‑saving keyboard shortcuts powered by GNU Readline for history and editing
